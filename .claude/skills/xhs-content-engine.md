# AI 内容工厂 Skill — 小红书自动化内容生成

一句话命令生成「AI 内容工厂」主题的小红书图文卡片。别人装了就能用，改自己的管道描述就行。

## 触发词

- "生成 AI 内容工厂卡片"
- "帮我做一套工具介绍的卡片"
- "把我的自动化管道做成小红书"

## 你会得到

- 6 张卡片 PNG（封面 + p1~p5），1080×1440 @2x
- 完整小红书发布文案（标题 + 正文 + 标签）
- 统一紫色 AI Skills 风格（`#2f4bf0`）

## 卡片结构

| 卡片 | 内容 | 目的 |
|------|------|------|
| cover | 双引擎预览 + 管道标签 + 共享基础设施 | 吸引点击 |
| p1 | 从手动到自动化（Before/After + 4 步方法论） | 建立共鸣 |
| p2 | 引擎一：你的核心管道深度拆解 | 展示能力 |
| p3 | 引擎二：你的辅助管道深度拆解 | 展示能力 |
| p4 | 对比表 + 共享基础设施 | 证明架构 |
| p5 | 3 步安装指南 | 降低门槛 |

## 使用方法

### 1. 准备你的管道描述

告诉我你的自动化管道，格式如下：

```
我的管道：
- 引擎一：[名称] — [一句话描述]
  步骤：①... → ②... → ③...
- 引擎二：[名称] — [一句话描述]
  步骤：①... → ②... → ③...

共享基础设施：[列出共享的组件]

安装步骤：
① ...
② ...
③ ...

小红书账号定位：[你的定位]
```

### 2. 生成卡片

说"生成卡片"，我会：
1. 用你的管道描述填充 6 张卡片 HTML（紫色 `#2f4bf0` 主题）
2. Chrome @2x 截图输出 2160×2880 PNG
3. 生成小红书文案（标题 + 正文 + #标签）

### 3. 发布

卡片在 `xiaohongshu/<date>/` 目录，直接上传小红书。

## 设计系统（内置，不需要你管）

```css
:root {
  --cream: #f4f2ee;    /* 页面底色 */
  --card: #ffffff;     /* 卡片底色 */
  --ink: #1a1a1a;      /* 主文字 */
  --soft: #6b6b6b;     /* 辅助文字 */
  --accent: #2f4bf0;   /* 强调色（紫色） */
  --line: #e7e4de;     /* 分割线 */
}
字体: PingFang SC, Inter
卡片尺寸: 1080×1440px, 截图 @2x = 2160×2880px
```

## 组件库（可选，按需拼装）

| 组件 | 用途 |
|------|------|
| `title-block` | 英文大标题 + 下划线 |
| `cn-block` | 中文副标题，关键词 accent 色高亮 |
| `compare-row` | 双栏对比（浅色 / accent 实色） |
| `steps-list` | 圆形编号 + 步骤文字 |
| `layers` | 编号圆 + 标题 + 灰色描述，适合层级架构 |
| `pipes` | 标签组，适合展示工具链 |
| `insight-blue` | accent 实色底 + 白色文字，核心洞察 |
| `insight-box` | 顶部分割线 + 金句 |
| `engines` | 双栏引擎预览（浅色 / accent 实色） |
| `reqs` | 编号列表，适合需求/依赖清单 |
| `code-block` | 深色底代码块，适合安装命令 |
| `choices` | 双栏选择卡片，适合路径对比 |

## 示例

用户说：
> 我的管道：引擎一是 AI 客服机器人，引擎二是自动数据分析。帮我生成卡片。

Claude 按上述 6 张卡片结构自动填充，生成 HTML + 渲染 PNG + 写文案。

## 自定义

- **颜色**：改 `--accent` 变量即可换主题色
- **卡片数**：默认 6 张（cover + p1~p5），可增减
- **语言**：默认中文，改标题和文案即可
- **管道数**：不一定两个引擎，一个或三个都行，调整 p2~p4 卡片内容即可
